Entry page for testing/examples.

Some rough cut tests to collect/display audio data, rip this code up to do what you need to do.

- steal from your friends.

You need to be on FireFox to get the audio to work. Chrome has 'issues' when it's not local and http (security).

Record Audio
Record Audio Stream, stream to Streams. This attaches to the browsers microphone.
If multiple laptops connect, you'll see them in the monitor page below.
Monitor Audio
Monitor the Audio that has been processed on streams. Multiple Streams are monitored from here.
Forward control
Send DEBUG Messages : Converse of Below, to get messages to Streams
Backward control
Send DEBUG Messages : Convert of Above, get messages from Streams

Notes:

  • This application sends message through a proxy to Streams. Due to security provided by BlueMix we cannot make a direct connection to Streams, thus the proxy
  • This proxy forwards data from the public internet to Streams which is residing in the protected lair of BlueMix. The proxy server needs the IP address of the Steams application. This is not as straight forward as I would like..
    Steps...
  1. Get the IP address the streams application.
  2. Get the configuration of the Streams application.
  3. Execute the curl command incorporating the data from the configuration
  4. The result will provide the IP mapping.
  5. The (recorder/monitor) pages have a input box put this value.
